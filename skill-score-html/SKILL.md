# 业务线测试任务总清单 → HTML 生成器

将标准格式的 `xxx-业务线测试任务总清单.xlsx` 转为可编辑、可筛选、可导出的单文件 HTML 页面。

## 输入

用户提供一个 xlsx 文件路径。

## Excel 标准格式（4 个 Sheet）

| Sheet | 内容 |
|---|---|
| 产品线全量测试任务表 | A-V 列：所属业务层、模块、测试类型、测试任务、任务类型、任务ID、任务名称、任务执行人、测试场景描述、测试集构成概述、测试环境、测试工具、统计指标及计算公式、合同指标、竞品指标、KA实际交付指标、5.0分、4.0分、3.0分、2.0分、1.0分、0.0分。HTML 表格额外增加"测试结果"和"判断机制标识"列（位于 0.0分 与操作列之间） |
| 任务权重定义 | 必测项/选测项权重 + 测试类型权重 |
| 模块权重定义 | 模块名 + 权重值 |
| 业务层权重定义 | L1/L2/L3 + 权重值 |

第 1 行为表头，第 2 行为评分等级行（5.0分~0.0分），第 3 行起为数据行。

## 执行步骤

1. 运行 `scripts/generate_html.py <xlsx文件路径>`，脚本会：
   - 读取 4 个 Sheet 的数据
   - 从文件名提取项目名（去掉 `-业务线测试任务总清单.xlsx` 后缀）
   - 生成与 xlsx 同目录、同名的 `.html` 文件
2. 用 `xdg-open` 打开生成的 HTML 文件供用户查看

## HTML 功能要求

生成的 HTML 页面必须包含以下功能：

### 表格展示
- 两栏 Tab 切换：「测试任务」和「权重定义」
- 表格列：#、所属业务层、模块、测试类型、测试任务、任务类型、任务ID、任务名称、测试场景描述、任务执行人、5.0分~0.0分、测试结果、判断机制标识、操作
- 模块、业务层、测试类型、测试任务、任务类型、任务ID 列强制单行显示（white-space: nowrap），列宽自适应（width: 1%）
- 任务名称、测试场景描述列自适应宽度
- 操作列强制单行显示，编辑按钮绿色、删除按钮橙色
- 测试结果列位于 0.0分 与判断机制标识之间，支持 contenteditable 行内直接编辑，也可通过模态框填写
- 判断机制标识列位于测试结果与操作之间，行内下拉选择，默认值为"正常评分（Normal）"，可选：正常评分（Normal）、锁定上限（Cap）、特批降级（Waived）、红线置零（Veto）、离散评分（SoftVeto）

### 筛选功能
- 5 个筛选维度：模块、测试类型、测试任务、任务类型（下拉选择）、任务ID（多选下拉，带全选）
- 任务ID 多选组件：勾选面板、全选/取消全选、已选数量显示、点击外部自动收起
- 任务ID 多选列表随其他筛选条件联动（选择模块/测试类型等后，任务ID 列表仅显示匹配项）
- 清除筛选按钮（同时清空任务ID多选）
- 筛选结果计数显示（必测/选测分别统计）

### 编辑功能
- 新增任务：弹出表单填写所有字段
- 编辑任务：点击行内编辑按钮弹出预填表单
- 删除任务：确认后删除
- 模态框表单使用 grid 布局，带 datalist 辅助输入

### 评分计算（集成）
- Tab 栏最右"评分计算"按钮，点击即使用当前全部任务数据（含已填测试结果和判断机制标识）直接计算
- 计算逻辑与 score_tool.html（质量体系评分计算工具 V3）一致，免去导出→上传的中间步骤
- 计算流程：原始评分→任务级评分→测试类型评分→模块级评分→层级评分→综合评分
- 支持 5 种判断机制：正常评分（Normal）、锁定上限（Cap）、特批降级（Waived）、红线置零（Veto）、离散评分（SoftVeto）
- 必测项 raw=2 固定为 Cap（C（Cap））或 Waived（B（Waived）），必测项 raw=1 固定为 Cap（D（Cap））
- 评分规则、标识格式与 score_tool.html 完全一致（全角括号）
- 结果展示：综合得分/评级摘要卡片 + 8 个子 Tab（红线项、任务级、测试类型、模块级、层级、综合、统计、缺失）

### 权重定义
- 4 个权重表（任务类型、测试类型、模块、业务层）分 2x2 展示
- 权重数值为 input 输入框，支持自定义修改

### 获取测试结果
- Tab 栏下方蓝色输入栏，支持 URL 获取（自动 CORS 代理回退）和本地文件上传两种方式
- 自动解析报告中的被测对象列表（如有多个显示下拉框供选择）
- 召回率列定位：解析报告 `<thead>` 表头，找最后一个"序号"th 作为数据列起点，再找其后的"召回率"th，计算相对列索引，用于提取 `env_results` 行的召回率值（兼容不同列数的报告格式）
- 匹配规则：
  - **Scene 编号**：报告场景名中的 Scene 编号（如 Scene11 → scene11、Scene1-1 → scene1-1、wkp-Scene07 → scene07）匹配 taskId
  - **task0* 编号**：报告场景名以 `task0*` 开头（如 task003_Noise-... → task003）直接匹配 taskId
  - 多匹配时用关键词（待机/噪声/安静/quiet + 单扫/扫拖）精确区分
- 功能测试项（测试类型为"功能测试"）未匹配到报告数据的，默认填入"1"
- 有效结果低于所有阈值时，原始评分返回 0（最低分）而非 NaN
- 自动将匹配到的召回率填入筛选后测试项的"测试结果"列

### 导出 Excel
- 点击"导出Excel"分别导出 3 个独立 xlsx 文件：
  1. **{项目名}-评分总表.xlsx**：全量数据（与页面筛选无关），所有列但不含"测试结果"和"判断机制标识"，两行表头（A-P 合并单元格，Q1 合并为"单个任务原始评分"标题），附带 3 个权重定义 Sheet
  2. **{项目名}-测试范围.xlsx**（按页面筛选结果）：所属业务层、模块、测试类型、测试任务、任务类型、任务ID、任务名称、任务执行人
  3. **{项目名}-项目实测数据.xlsx**（按页面筛选结果）：所属业务层、模块、测试类型、测试任务、任务类型、任务ID、任务名称、测试结果、判断机制标识
- 使用 SheetJS（xlsx）库，CDN 引入

### 样式规范
- 蓝色渐变 header，标签 tag 区分业务层/任务类型
- 绿色编辑按钮、橙色删除按钮
- 表格 sticky header、斑马纹、hover 高亮
- 响应式滚动表格区域
